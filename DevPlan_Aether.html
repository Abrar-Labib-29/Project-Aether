1 Tech Stack

Every choice below was made against a single constraint: this must run invisibly on a 3 GB Android 9 device forever. No convenience libraries. No heavy frameworks. Pure Android SDK.

🇰

Kotlin (100%)

Modern, null-safe, concise. No Java. Kotlin stdlib is already in Android runtime — zero extra APK weight for modern syntax.

🏗️

Android SDK Only

Zero third-party dependencies. No Retrofit, Dagger, Glide, or any library. Pure AOSP APIs.

⚙️

Foreground Service

The backbone of persistence. START_STICKY + BOOT receiver. Cannot be killed by LMK.

🪟

WindowManager Overlay

TYPE_APPLICATION_OVERLAY draws above everything. No Activity needed after startup.

📷

MediaProjection API

Only valid screenshot method on API 28. Token cached per session. ImageReader + VirtualDisplay.

🔊

AudioManager

adjustVolume() with FLAG_SHOW_UI. Native volume OSD appears — zero custom UI needed for volume.

🎨

Canvas Drawing

Bubble drawn with Canvas.drawCircle(). No image assets. No layout inflation overhead.

🛠️

Build: Gradle + AGP

Android Gradle Plugin. No R8 full-mode — just minification + shrinking. Release build, debug-signed locally.

🚫
Explicitly Rejected Libraries

Hilt/Dagger (DI overhead), Retrofit (no network), Room (no database), Glide/Coil (no image loading), Coroutines-heavy patterns (simple Handler is enough), Jetpack Compose (too heavy for a service-rendered view), any analytics SDK, any ads SDK.

2 Project File Structure

Keep the structure dead flat. Every file you create should have exactly one clear job.

Aether/ ├── app/ │ ├── src/main/ │ │ ├── java/com/Aether/ │ │ │ ├── LauncherActivity.kt ← Permission gate, starts service, finishes │ │ │ ├── AetherService.kt ← Foreground service, manages all views │ │ │ ├── BubbleView.kt ← Custom View: drag, snap, opacity │ │ │ ├── ActionPanelView.kt ← 3-button panel, animations │ │ │ ├── ScreenshotManager.kt ← MediaProjection, ImageReader, save │ │ │ └── BootReceiver.kt ← BOOT_COMPLETED → startForegroundService │ │ ├── res/ │ │ │ ├── drawable/ │ │ │ │ ├── ic_vol_up.xml ← Vector: volume up icon │ │ │ │ ├── ic_vol_down.xml ← Vector: volume down icon │ │ │ │ └── ic_screenshot.xml ← Vector: camera/screenshot icon │ │ │ ├── values/ │ │ │ │ ├── colors.xml │ │ │ │ └── strings.xml │ │ │ └── xml/ │ │ │ └── file_paths.xml ← FileProvider paths for media │ │ └── AndroidManifest.xml │ └── build.gradle ← minSdk 28, no external deps └── build.gradle ← Project-level Gradle
📁
6 Kotlin files. That's it.

The entire app is 6 Kotlin files + 3 vector drawables + the manifest. This is intentional. More files = more complexity = harder to debug in a one-day build. Keep it flat and simple.

3 Phase-by-Phase Build Plan

9:00 – 10:30 AM
Phase 0 — Environment Setup & Project Scaffold
~1.5 hrs

Get Android Studio ready and the project compiling to an empty APK before writing any feature code.

  • Install Android Studio (Hedgehog or newer). During setup, install Android SDK Platform 28 specifically.
    CRITICAL
  • Create new project: Empty Activity, package com.Aether, Language: Kotlin, Min SDK: API 28.
    CRITICAL
  • In build.gradle (app): set minSdk = 28, targetSdk = 34, compileSdk = 34. Remove all template dependencies except core-ktx and appcompat.
    CRITICAL
  • Enable R8 shrinking: in release buildType set minifyEnabled = true, shrinkResources = true. This cuts APK size dramatically.
    IMPORTANT
  • Write AndroidManifest.xml fully (see manifest spec in Section 4 below). Declare all permissions, service, receiver, activity.
    CRITICAL
  • Enable USB debugging on your Walton RX7 Mini. (Settings → About → tap Build Number 7 times → Developer Options → USB Debugging). Connect via USB.
    CRITICAL
  • Run a blank build to confirm the project compiles and deploys to the device. If this works, the scaffold is done.
    NORMAL
10:30 AM – 1:00 PM
Phase 1 — Core Service & Floating Bubble
~2.5 hrs

Build the survival engine. By the end of this phase, a draggable bubble must be permanently floating on screen and survive being sent to the background.

Step 1 — LauncherActivity.kt

This Activity has no layout. Its only job is to check the overlay permission and start the service, then immediately call finish().

  • Check Settings.canDrawOverlays(this). If false, launch ACTION_MANAGE_OVERLAY_PERMISSION intent and wait for onActivityResult.
    CRITICAL
  • If permission granted, call startForegroundService(Intent(this, AetherService::class.java)).
    CRITICAL
  • Call finish() immediately so no Activity stays in the task stack.
    CRITICAL

Step 2 — AetherService.kt

  • Extend Service. In onStartCommand() return START_STICKY.
    CRITICAL
  • Create notification channel (IMPORTANCE_LOW) and call startForeground(1, notification) as the very first action. This is what keeps the service alive.
    CRITICAL
  • Get WindowManager via getSystemService(WINDOW_SERVICE).
    CRITICAL
  • Create WindowManager.LayoutParams with type TYPE_APPLICATION_OVERLAY, flags FLAG_NOT_FOCUSABLE or FLAG_LAYOUT_IN_SCREEN, format TRANSLUCENT.
    CRITICAL
  • Instantiate BubbleView and call windowManager.addView(bubbleView, layoutParams).
    CRITICAL
  • In onDestroy() remove the view: windowManager.removeView(bubbleView). Also call stopForeground(true).
    IMPORTANT

Step 3 — BubbleView.kt

  • Extend View. Override onDraw(canvas: Canvas) — draw a filled circle with canvas.drawCircle(). No bitmap loading. Use Paint only.
    CRITICAL
  • Override onTouchEvent(event: MotionEvent). Track ACTION_DOWN, ACTION_MOVE, ACTION_UP.
    CRITICAL
  • On ACTION_MOVE: update WindowManager.LayoutParams.x and .y, call windowManager.updateViewLayout(this, params).
    CRITICAL
  • On ACTION_UP: calculate which edge (left or right) is closer. Animate params.x to 0 or screenWidth - bubbleSize using a ValueAnimator with OvershootInterpolator.
    IMPORTANT
  • Distinguish tap vs drag by comparing total movement to a threshold of 8dp. If movement < 8dp on ACTION_UP, call onTapListener().
    IMPORTANT
  • Idle fade: post a Runnable on a Handler(Looper.getMainLooper()) with 3000ms delay. On fire: animate alpha from 1.0f to 0.18f. Cancel + repost on any touch event.
    IMPORTANT

✅ Checkpoint: At this point, deploy to the device. The bubble should appear, be draggable, snap to the edge, and fade after 3 seconds. Open another app — the bubble should remain. Lock and unlock the phone — the bubble should still be there.

1:00 PM – 4:00 PM
Phase 2 — Volume Control, Screenshot & Boot Receiver
~3 hrs

Build all three functional features. Volume is straightforward. Screenshot requires careful MediaProjection handling.

Step 4 — ActionPanelView.kt

  • Extend LinearLayout (vertical). Inflate or programmatically add three ImageButtons: Vol+, Vol-, Screenshot. Set background to a dark rounded card via a shape drawable.
    IMPORTANT
  • Add show/hide methods using ViewPropertyAnimator: scaleX/Y from 0.7→1.0 + alpha 0→1 for show (150ms), reverse for hide (120ms).
    IMPORTANT
  • Add the panel as a second WindowManager view in AetherService. Position it adjacent to the bubble (left if bubble is on right edge).
    IMPORTANT
  • Wire up Vol+ button → audioManager.adjustVolume(AudioManager.ADJUST_RAISE, AudioManager.FLAG_SHOW_UI).
    CRITICAL
  • Wire up Vol- button → audioManager.adjustVolume(AudioManager.ADJUST_LOWER, AudioManager.FLAG_SHOW_UI).
    CRITICAL

Step 5 — ScreenshotManager.kt

ℹ️
MediaProjection Flow on API 28

You cannot request MediaProjection from a Service directly. You need to launch a transparent Activity that requests the permission, receives the result in onActivityResult, and passes the Intent back to the Service. This is the correct, mandatory architecture for API 28.

  • Create ScreenCaptureRequestActivity — a transparent, no-animation Activity. In onCreate(), get MediaProjectionManager and call startActivityForResult(manager.createScreenCaptureIntent(), REQUEST_CODE).
    CRITICAL
  • In onActivityResult, pass the result Intent back to AetherService via a local broadcast (LocalBroadcastManager or a direct reference). Call finish().
    CRITICAL
  • In ScreenshotManager: store the MediaProjection object. Create ImageReader with screen size and PixelFormat.RGBA_8888. Create VirtualDisplay from the projection.
    CRITICAL
  • Before capture: call hideOverlay() on both views (set alpha to 0f instantly, no animation). Wait one frame (post to Handler) before capturing to ensure the overlay is fully hidden in the next render frame.
    CRITICAL
  • Acquire the latest Image from ImageReader. Read the byte buffer. Create a Bitmap. Release the Image immediately to free the buffer.
    CRITICAL
  • Save the Bitmap to MediaStore.Images using ContentValues. Set MIME type to image/png, relative path to Pictures/Screenshots, display name with timestamp.
    CRITICAL
  • After save: call showOverlay() to restore view alpha. Show Toast.makeText("Screenshot saved").
    IMPORTANT
  • Do ALL file I/O and Bitmap processing on a background thread (Thread { }.start() is fine — no need for coroutines for this). Only call UI operations back on the main thread.
    CRITICAL

Step 6 — BootReceiver.kt

  • Extend BroadcastReceiver. In onReceive(): check action == Intent.ACTION_BOOT_COMPLETED, then call context.startForegroundService(Intent(context, AetherService::class.java)).
    CRITICAL
  • Register it in the manifest with <action android:name="android.intent.action.BOOT_COMPLETED"/> and android:exported="true".
    CRITICAL

✅ Checkpoint: Volume buttons should adjust media volume with the system OSD appearing. Screenshot button should capture the screen (without the bubble visible) and the image should appear in Gallery.

4:00 PM – 6:00 PM
Phase 3 — UI Polish & Edge Cases
~2 hrs

The app works. Now make it feel good. Fix edge cases that will annoy you daily.

  • Persist bubble position. Save last X/Y position to SharedPreferences on every drag-release. Restore position on service start. So the bubble stays where you last left it after reboot.
    IMPORTANT
  • Keyboard avoidance. Add flag FLAG_NOT_TOUCH_MODAL to WindowManager params so the overlay does not steal keyboard focus from other apps.
    CRITICAL
  • Safe zones. Clamp the bubble's Y position so it cannot go behind the status bar (top ~24dp) or navigation bar (bottom ~48dp).
    IMPORTANT
  • Panel position correction. When the panel would extend off-screen (bubble near top or bottom), flip the panel's layout direction. Clamp its Y so it stays within screen bounds.
    IMPORTANT
  • Long-press volume. Add setOnLongClickListener on Vol+/Vol- buttons. Use a Handler.postDelayed() repeating every 300ms to repeatedly call adjustVolume() until the finger lifts.
    NORMAL
  • Notification "Stop" action. Add an action to the notification that broadcasts a stop intent. In the service, register a receiver for this intent and call stopSelf().
    IMPORTANT
  • Outside-tap dismiss. Add a full-screen transparent view below the panel in the WindowManager. When tapped, dismiss the panel and remove this view.
    NORMAL
  • Launcher icon. Add a simple launcher icon. The app icon should be visible so the user can restart Aether if needed.
    NORMAL
  • First-run flow. If SYSTEM_ALERT_WINDOW permission was just granted, show a brief Toast: "Aether is running. Look for the floating button."
    NORMAL

Most Common Bugs to Watch For

Bubble appears in screenshot (hide timing issue) · Volume change crashes if audio focus not available (use try-catch) · Service gets killed (check startForeground is called before any other operation) · Panel goes off-screen near bottom (clamping bug) · App crashes on first permission grant because SYSTEM_ALERT_WINDOW check happens before the settings screen returns.

Phase 5 — Release Build & Install
~1.5 hrs

Build the final APK, verify it is under 2.5 MB, sign it, and install it permanently on the device.

  • In Android Studio: Build → Generate Signed Bundle/APK → APK. Create a new keystore (keep it safe). Build Release variant.
    CRITICAL
  • Check APK size. Go to Build → Analyze APK. Should be < 2.5 MB. If larger, check for leftover resources or accidental dependencies.
    IMPORTANT
  • Uninstall the debug APK from the device first to avoid signature conflicts: adb uninstall com.Aether
    CRITICAL
  • Install the release APK: adb install app-release.apk
    CRITICAL
  • Run the full test checklist (T01–T15) one more time on the release build. Release builds behave differently — R8 can sometimes strip needed classes if ProGuard rules are missing.
    IMPORTANT
  • Reboot the device once more and confirm bubble auto-starts via BootReceiver.
    IMPORTANT
  • Copy the signed APK to a safe location (Google Drive, USB, SD card) as a backup — so you can reinstall if needed without rebuilding.
    NORMAL

4 AndroidManifest.xml Reference

The manifest is one of the most critical files. Get this wrong and nothing works. Here is the exact structure you need:

<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.Aether">

    <!-- Auto-granted permissions -->
    <uses-permission android:name="android.permission.FOREGROUND_SERVICE"/>
    <uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED"/>
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>

    <!-- Special permission: user must grant manually in Settings -->
    <uses-permission android:name="android.permission.SYSTEM_ALERT_WINDOW"/>

    <application
        android:allowBackup="false"
        android:icon="@mipmap/ic_launcher"
        android:label="Aether"
        android:theme="@style/Theme.AppCompat.NoActionBar">

        <!-- Entry point: no UI, just permission gate -->
        <activity
            android:name=".LauncherActivity"
            android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>
                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity>

        <!-- Transparent activity for MediaProjection permission request -->
        <activity
            android:name=".ScreenCaptureRequestActivity"
            android:exported="false"
            android:theme="@android:style/Theme.Translucent.NoTitleBar"/>

        <!-- The main survival engine -->
        <service
            android:name=".AetherService"
            android:exported="false"
            android:foregroundServiceType="mediaProjection"/>

        <!-- Starts service on device boot -->
        <receiver
            android:name=".BootReceiver"
            android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.BOOT_COMPLETED"/>
            </intent-filter>
        </receiver>

    </application>
</manifest>

Definition of Done

The build is complete when: all T01–T11 tests pass on the release APK, the APK is under 2.5 MB, the service survives 30 minutes in the background, boot auto-start works after a reboot, and the signed APK is backed up. Everything else is polish for v2.
